# 基于多元大数据的佛山市南海区文旅融合调研

## 中期阶段研究方法论报告

------

## 一、研究总体框架

### 1.1 技术路线概述

本研究采用"数据采集 → 清洗结构化 → 双谱系构建 → 耦合分析 → 空间分析 → 知识图谱可视化"的六阶段技术路线。每一阶段的输入、方法和输出关系如下：

```
文化典籍(43份) ──OCR/文本整理──→ 结构化文本(3,271,537字)
                                      │
                                      ├──jieba分词+NER──→ 文化实体(2,279个)+关系(5,000条)
                                      │                        │
                                      │                        ├──→ 文化谱系树(8大类97条目)
                                      │                        │
高德API ──POI检索──→ 1,353条POI       │                        │
                        │              ├──→ 知识图谱(150节点224关系)
                        ├──清洗分类──→ 标准化POI               │
                        │              │                        │
评论爬取(948条) ──情感分析──→ 体验度   ├──→ 景点谱系树(1,353个)  │
                                      │                        │
高德API ──行政区查询──→ 精确边界       ├──→ 耦合分析(强16/错9/缺A14/缺B6)
                                      │
高德API ──地理编码──→ 非遗坐标(26项)  ├──→ 空间分析(核密度+聚类+镇街对比)
```

### 1.2 研究方法体系

| 方法类别 | 具体方法 | 应用环节 |
|----------|----------|----------|
| 文本分析 | OCR识别、jieba中文分词、正则实体识别、TF-IDF权重、共现关系抽取 | 典籍处理、NER |
| 空间分析 | 核密度估计(KDE)、DBSCAN空间聚类、Haversine距离、地理编码 | 空间分析 |
| 图谱方法 | 实体-关系建模、力导向图布局、共现网络 | 知识图谱 |
| 评价方法 | 多指标加权评分、情感分析、层次分级 | 体验度评估 |
| 对比方法 | 实体对齐、关系映射、耦合状态判定 | 双谱系耦合 |

## 二、数据采集环节

### 2.1 古籍OCR整合（consolidate_ocr.py）

**研究目的：** 将看典古籍平台OCR产出的458页散落文本合并为可分析的连续文档。

**方法：**
1. 遍历`古籍识别/`目录下所有`*_换行.txt`文件
2. 按页码数字排序（从文件名提取数字，如`401_换行.txt` → 页码401）
3. 合并为两种格式：
   - 带页码标记的全文（保留`--- 第N页 ---`分隔符，便于溯源）
   - 去除页码的连续文本（便于NLP处理）
4. 生成页码索引JSON（记录每页的字符偏移量，支持从实体定位到原始页面）
5. 同步整理开题阶段22份文化典籍MD文件，统一命名存放

**结果：**
- 南海县志OCR全文：375,453字符（458页）
- 22份典籍整理到统一目录
- 18个有效文本源，总计3,271,537个中文字符

**推理依据：** 原始OCR数据以单页为单位存储，无法直接进行跨页的文本分析（如句子可能跨页断裂）。合并后的连续文本才能支持实体共现关系的准确提取。

------

### 2.2 高德POI真实采集（amap_real_data.py）

**研究目的：** 获取南海区全部文旅相关兴趣点的真实数据，包括精确坐标、类型、评分等。

**方法：**
1. **按类型编码检索**（15种类型）：旅游景点(110000)、风景名胜(110100)、公园广场(110101)、纪念馆(110205)、历史建筑(110206)、博物馆(140100)、展览馆(140200)、图书馆(140300)、科技馆(140400)、文化宫(140500)、影剧院(140600)、文化传媒(141200)、宗教场所(160000)、体育休闲(080000)等
2. **按关键词检索**（18个关键词）："非遗"、"传习所"、"传承基地"、"古村"、"古镇"、"祠堂"、"庙宇"、"纪念馆"、"文化馆"、"遗址"、"醒狮"、"龙舟"、"武术"、"咏春"、"书院"、"故居"、"古建筑"、"历史建筑"
3. 每次请求限定`city=佛山`、`citylimit=true`，返回结果中二次过滤`adname`含"南海区"
4. 分页采集（每页25条，最多翻20页），请求间隔0.3秒避免触发频控
5. 按高德POI ID去重

**API参数：**
- 接口：`https://restapi.amap.com/v3/place/text`
- 扩展信息：`extensions=all`（获取评分、照片等详细字段）
- Key：高德Web服务API Key

**结果：**
- 原始返回：1,590条
- 去重后：**1,353条**真实POI
- 包含字段：名称、高德ID、类型、类型编码、地址、省/市/区、经纬度、电话、评分(rating)、人均消费(cost)、照片数量

**推理依据：** 采用"类型+关键词"双重检索策略，是因为高德的类型编码体系无法完全覆盖文旅相关POI。例如"祠堂"在高德中没有独立类型编码，必须通过关键词补充检索。两种方式互补后去重，可最大化覆盖率。

------

### 2.3 行政区划精确边界（amap_real_data.py）

**研究目的：** 获取南海区及下辖7个镇街的精确行政边界，用于空间分析的底图。

**方法：**
1. 调用高德行政区域查询API：`https://restapi.amap.com/v3/config/district`
2. 第一步查询`keywords=南海区`，`subdistrict=1`获取区级边界和下辖镇街列表
3. 第二步逐个查询每个镇街的adcode，获取镇街级边界
4. 将高德返回的polyline字符串（`lng,lat;lng,lat|lng,lat;lng,lat`格式）解析为GeoJSON标准坐标
5. 处理多环多边形（`|`分隔的多个环）

**结果：**
- 南海区整体边界 + 7个镇街边界 = 8个GeoJSON Feature
- 7个镇街中心点坐标：桂城(113.141,23.024)、西樵(112.883,22.849)、九江(113.036,22.837)、狮山(113.083,23.248)、里水(113.198,23.170)、丹灶(112.943,23.118)、大沥(113.117,23.136)

**推理依据：** 空间分析需要精确的行政边界作为底图，手绘简化多边形会导致POI的镇街归属判断错误，进而影响镇街资源密度统计的准确性。

------

### 2.4 非遗项目精确地理编码（amap_real_data.py）

**研究目的：** 将26个非遗项目从"名称+所属镇街"转化为精确经纬度坐标。

**方法：**
1. 构建搜索地址：`佛山市南海区` + 非遗名称关键词 + 所属镇街
2. 优先使用地理编码API：`https://restapi.amap.com/v3/geocode/geo`
3. 若地理编码无结果，回退使用POI搜索API按关键词查找
4. 双重策略确保每个项目都能获得坐标

**结果：**
- 26/26个非遗项目全部获得精确坐标
- 编码来源：高德地理编码API + 高德POI搜索（回退补充）

**推理依据：** 非遗项目多数不是标准地名（如"狮舞（广东醒狮）"），直接地理编码可能失败。通过"地理编码优先+POI搜索回退"的双重策略，确保所有项目都能定位。

------

### 2.5 真实评论爬取（review_crawler_real.py）

**研究目的：** 获取南海区主要景点的真实用户评价数据，用于体验度分析。

**方法：**
1. **高德评分提取**：从已采集的1,353条POI中提取所有非零评分，作为评价基础数据
2. **携程评论爬取**：对有携程景点ID的景点，调用携程移动端评论API获取文字评论
   - 接口：`https://m.ctrip.com/restapi/soa2/13444/json/getCommentCollapseList`
   - 每个景点最多采集3页（30条）
3. **马蜂窝尝试**：请求景点详情页并用正则提取评论文本（因反爬机制获取量有限）
4. **情感分析**：基于正面词表（好/美/棒/赞/推荐/值得等25词）和负面词表（差/烂/失望/贵等16词）进行简单情感分类

**结果：**
- 高德评分：917条（含评分数值，无文字）
- 携程评论：31条（含评分+完整文字评论）
- 马蜂窝：0条（反爬严格）
- 总计：**948条**真实评论/评分数据

**推理依据：** 大众点评反爬最强，携程移动端API相对开放，高德虽无文字评论但评分覆盖面广。三源互补策略可在合规前提下获取最多数据。

## 三、数据清洗与结构化环节

### 3.1 NER文化实体抽取（text_ner.py）

**研究目的：** 从3,271,537字的典籍文本中自动提取文化实体（人物、地点、建筑、文化要素等），形成结构化知识。

**方法：**
1. **jieba分词**：使用jieba.cut对每个文本源进行中文分词
2. **自定义词典**：预定义6类共150+个领域关键词（人物29个、地点40个、建筑30个、文化要素35个、事件16个、朝代18个），通过`jieba.add_word`注入，确保专业术语不被错误切分
3. **实体类型判定**（优先级从高到低）：
   - 自定义词典精确匹配 → 置信度0.95
   - 后缀规则判定（如`*村/镇/区`→地点，`*祠/庙/院`→建筑，`*拳/舞/曲`→文化要素）→ 置信度0.75~0.8
   - 频次过滤：未匹配到类型且频次<5的词被过滤
4. **TF-IDF计算**：对每个实体计算TF（词频/总词数）× IDF（log(总词数/该词频次)），衡量实体在文本中的重要程度
5. **多源合并**：来自多个典籍的同名实体合并，跨源权重 = 频次 × (1 + 0.3 × (出现源数-1))，出现在越多典籍中的实体权重越高
6. **共现关系提取**：将文本按句号/感叹号/问号/分号/换行分句，统计同一句子中共同出现的实体对，共现次数≥2的保留为关系

**结果：**
- 文化实体：**2,279个**（地点158、建筑163、其他1726、文化要素83、时间57、朝代21、组织5、事件2）
- 共现关系：**5,000条**（取共现频次最高的前5000对）

**推理依据：** 本研究不使用深度学习NER模型（如BERT-NER），原因是：(1)文旅领域缺少标注训练数据；(2)OCR文本含噪声，深度模型易过拟合噪声。而jieba分词+自定义词典+后缀规则的组合方案，虽精度低于深度模型，但在零标注数据条件下是最实用的选择。"其他"类占比高（75.7%）是主要不足，后续可通过扩充词典和引入上下文规则优化。

------

### 3.2 POI数据清洗与标准化（poi_cleaner.py）

**研究目的：** 将1,353条原始POI统一格式、自动分类、关联非遗项目，形成可分析的标准化数据集。

**方法：**
1. **自动分类**（9类）：根据POI名称和高德类型字段，基于关键词匹配判定类别
   - 自然景观：含"风景/公园/山/湖/岛"等 → 677条
   - 文化场馆：含"博物馆/纪念馆/展览馆/文化馆/图书馆"等 → 154条
   - 体育设施：含"体育/运动"等 → 128条
   - 非遗体验：含"非遗/传习所/传承基地/武术馆"等 → 19条
   - 公园绿地、宗教场所、人文古迹、名人故居、特色街区等
2. **非遗关联匹配**：定义15组非遗关键词（如"醒狮"→["醒狮","狮舞","龙狮"]），对每个POI的名称+类型进行关键词扫描
3. **镇街归属判定**：优先根据地址关键词（如"桂城"→桂城街道），其次根据经纬度范围粗判
4. **评分标准化**：将字符串评分转为浮点数，空值记为0

**结果：**
- 标准化POI：**1,353条**
- 分类统计：自然景观677、其他353、文化场馆154、体育128、非遗19、公园13、人文5、故居3、宗教1
- 镇街分布：桂城327、里水254、大沥234、狮山232、丹灶131、西樵128、九江47
- 非遗关联：**97个POI**与非遗项目建立了关联（关联率7.2%）

**推理依据：** 高德的类型编码体系与本研究的文旅分类体系不一致（高德无"非遗体验""人文古迹"等分类），因此需要基于关键词的二次分类。非遗关联率仅7.2%说明绝大部分景点与非遗之间缺少显性联系，这本身就是文旅融合不足的量化证据。

## 四、双谱系构建环节

### 4.1 文化谱系构建（culture_genealogy.py）

**研究目的：** 基于典籍文本和非遗名录，构建南海区文化的层级分类体系，揭示文化传承脉络。

**方法：**
1. **知识来源**：综合以下三类信息构建分类：
   - NER提取的2,279个文化实体（量化依据）
   - 南海区90项非遗名录（权威分类）
   - 文化典籍的章节结构和主题（定性参考）
2. **分类维度**：
   - 门类维度：按文化类型分为8大类（武术、饮食、建筑、民俗、工艺、音乐戏曲、学术、中医药）
   - 时间维度：标注每个子类的时间跨度（如"明代至今"）
   - 人物/场所维度：标注每个子类的代表人物和关键地点
3. **层级结构**：大类 → 子类 → 具体条目，同时在叶节点下挂载"代表人物"分支
4. **ECharts树形图可视化**：使用LR（左到右）布局，初始展开2层，支持交互展开/折叠

**结果：**
- **8个大类**：武术文化、饮食文化、建筑文化、民俗文化、工艺文化、音乐戏曲、学术文化、中医药文化
- **24个子类**：醒狮龙舞、传统拳术、龙舟竞渡、酿造技艺、传统食品、古村落、祠堂建筑、书院文化、宗教建筑、节庆活动、编织技艺、金属工艺、纺织工艺、岭南理学、科举传统等
- **97个具体条目**
- **15个关键人物**：黄飞鸿、康有为、叶问、湛若水、方献夫等
- **8个关键地点**：西樵镇、九江镇、桂城街道等

**推理依据：** 文化谱系的分类逻辑以非遗名录的官方分类为骨架（保证权威性），以典籍内容补充历史纵深（保证学术性），以NER实体频次验证重要性（保证数据驱动性）。三者互校：若某文化要素在典籍中高频出现但不在非遗名录中（如"书院文化"），说明这是历史上重要但当前保护体系未覆盖的文化要素。

------

### 4.2 景点谱系构建（scenic_genealogy.py）

**研究目的：** 基于真实POI和评论数据，构建现实文旅产品的层级结构，并量化每个景点的体验度。

**方法：**
1. **数据输入**：1,353条清洗后POI + 867条景点评论汇总
2. **层级结构**：按 类别 → 体验度等级(高/中/低) → 具体景点 三层构建
3. **体验度评估模型**（五维加权评分）：

   | 维度 | 权重 | 计算方法 | 满分条件 |
   |------|------|----------|----------|
   | 平台评分(score_rating) | 30% | avg_rating / 5.0 × 100 | 评分5.0 |
   | 好评率(score_positive) | 25% | 正面评论占比 | 100%好评 |
   | 文化深度(score_culture) | 20% | has_nh×40 + nh_count×20, max=100 | 关联3+个非遗 |
   | 评论活跃度(score_review) | 15% | review_count / 30 × 100, max=100 | 30+条评论 |
   | 照片丰富度(score_photos) | 10% | photo_count / 15 × 100, max=100 | 15+张照片 |

   总分 = 0.30×评分 + 0.25×好评率 + 0.20×文化深度 + 0.15×活跃度 + 0.10×照片

4. **分级标准**：≥70分=高，≥45分=中，<45分=低
5. **镇街分布树**：按 镇街 → 类别 → 景点 构建另一棵树

**结果：**
- 体验度分级：高=0、中=483、低=870
- 高级为0的原因：评估标准要求同时满足高评分+丰富评论+强文化关联，目前大部分景点仅在部分维度达标
- 镇街视角下，桂城街道资源最密集（327个），九江镇最少（47个）

**推理依据：** 五维指标的权重设定体现"文旅融合"研究视角——文化深度占20%（体现文化维度的重要性），而非纯粹的旅游评分。这使得与非遗关联的景点能获得额外加分，从而识别出"文化加持"型优质景点。

------

## 五、知识图谱与融合分析环节

### 5.1 知识图谱扩展（knowledge_graph.py）

**研究目的：** 整合文化实体、景点、非遗项目和空间信息，构建跨域关联的文旅知识图谱。

**方法：**
1. **节点来源**（150个节点）：
   - 文化实体TOP80（按跨源权重排序取前80个高频实体）
   - 景点TOP30（从1,353条POI中选取前30个）
   - 全部26个非遗项目
   - 7个镇街 + 8个文化门类
2. **关系来源**（224条边）：
   - 文化实体共现关系（从5,000条中筛选两端均在节点集中的前300条）
   - POI-镇街"位于"关系
   - POI-非遗"文化承载"关系（基于poi_cleaner的关键词匹配）
   - 非遗-镇街"传承于"关系
   - 文化门类-非遗"包含"关系（基于文化谱系分类）
3. **节点大小**：size = min(40, max(12, 10 + sqrt(mentions) × 1.5))
4. **可视化**：ECharts力导向图布局，11种颜色区分实体类型

**结果：**
- 150个节点、224条关系
- 类型分布：其他68、景点29、非遗26、地点8、文化门类8、镇街7、朝代2、人物1、建筑1、时间1

**推理依据：** 知识图谱的核心价值在于跨域关联——文化实体与景点之间的连接是通过"非遗项目"和"镇街"作为中介实现的。这种"文化层-中介层-景点层"的三层结构正是开题报告中提出的"文化载体资源"概念的图谱化表达。

### 5.2 双谱系耦合分析（coupling_analysis.py）

**研究目的：** 系统对比文化谱系与景点谱系，识别文旅融合的成功案例、不足之处和潜力方向。

**方法：**
1. **实体对齐**：人工建立25对文化要素-景点的映射关系（基于以下四种匹配类型）
   - 实体对应：非遗项目名 ↔ 传承基地/博物馆名（如"九江双蒸酒"↔"九江双蒸博物馆"）
   - 文化承载：文化人物/概念 ↔ 纪念场所（如"黄飞鸿武术文化"↔"黄飞鸿纪念馆"）
   - 空间共存：文化遗存 ↔ 所在景区（如"岭南理学/书院文化"↔"西樵山书院遗址"）
   - 主题关联：文化主题 ↔ 主题景点（如"宗教信仰文化"↔"南海观音寺"）
2. **耦合状态判定**：
   - 强耦合（文化有+景点有=融合成功）：实体对应、文化承载类 → **16对**
   - 错位（关联弱+体验浅=挖掘不足）：空间共存、主题关联类 → **9对**
   - 缺失A（文化有+景点无=未转化）：遍历非遗名录中无对应旅游产品的项目 → **14项**
   - 缺失B（文化无+景点有=有形无魂）：遍历景点中无文化内涵支撑的现代设施 → **6个**
3. **优化建议生成**：基于缺失分析，提出5条针对性建议

**结果：**
- 强耦合16对：主要集中在非遗传承基地（如醒狮、咏春、十番音乐等都有专门基地）
- 错位9对：书院文化、古村落等虽有空间载体但体验化程度低
- 缺失A14项：洪拳、煎堆技艺、鱼花习俗、灰塑、竹编等省级/市级非遗尚无旅游化载体
- 缺失B6个：千灯湖、映月湖等现代公园缺少文化灵魂

**推理依据：** 耦合分析的核心逻辑是——文化资源是"供给侧"（典籍记载的文化遗产），景点是"需求侧"（游客可体验的旅游产品），两者的匹配程度决定了文旅融合水平。"缺失A"（文化有但景点无）是最大的开发潜力所在；"缺失B"（景点有但文化无）是最迫切需要内容注入的对象。

## 六、空间分析环节（spatial_analysis.py）

**研究目的：** 分析文旅资源的空间分布特征，识别热点区域和发展不均衡问题。

### 6.1 核密度分析

**方法：**
- 网格化：将南海区范围划分为40×40=1,600个网格单元
- 核函数：Epanechnikov核 K(u) = (1-u²)² (u<1时)
- 带宽：3.0km（POI）和3.0km（非遗）
- 距离计算：Haversine公式（考虑地球曲率）
- 归一化：密度值除以最大值，映射到[0,1]

**结果：** POI热力集中在桂城-大沥-狮山中心城区带；非遗热力集中在西樵-九江-丹灶西南文化带。两者呈错位分布。

### 6.2 DBSCAN空间聚类

**方法：**
- 输入：1,353个POI坐标 + 26个非遗坐标 = 1,379个点
- 参数：eps=3.0km（邻域半径），min_points=3（最少核心点数）
- 距离度量：Haversine球面距离

**结果：** 识别2个聚类（中心城区集聚带、西南文化带），0个噪声点

### 6.3 镇街资源密度对比

**方法：** 统计每个镇街的POI数量和非遗数量，分别判定文化密度（非遗≥5=高，≥2=中）和旅游密度（POI≥8=高，≥4=中）

**结果：**
| 镇街 | POI数 | 非遗数 | 文化密度 | 旅游密度 | 融合特征 |
|------|-------|--------|----------|----------|----------|
| 桂城街道 | 327 | 7 | 高 | 高 | 综合最强 |
| 里水镇 | 254 | 2 | 中 | 高 | 旅游强于文化 |
| 大沥镇 | 234 | 4 | 中 | 高 | 旅游强于文化 |
| 狮山镇 | 232 | 2 | 中 | 高 | 旅游强于文化 |
| 丹灶镇 | 131 | 5 | 高 | 高 | 文化底蕴深 |
| 西樵镇 | 128 | 1 | 低 | 高 | 文旅名片但非遗归属少 |
| 九江镇 | 47 | 5 | 高 | 中 | **文化富集但旅游不足** |

**核心发现：** 九江镇是典型的"文化富集-旅游洼地"，非遗5项但POI仅47个，是文旅融合开发的首要潜力区域。

## 八、自省与优化方向

### 8.1 数据层面的不足

1. **NER"其他"类占比过高（75.7%）**：2,279个实体中有1,726个被标记为"其他"，说明实体类型判定规则覆盖面不足。后续应扩充领域词典（如增加南海区所有自然村名、历史人物名、传统技艺名等），或引入HanLP的命名实体识别模块进行二次校验。

2. **携程文字评论仅31条**：反爬机制限制了评论采集量。优化方向：(1)使用Selenium模拟浏览器绕过反爬；(2)尝试更多数据源（如去哪儿、百度旅游）；(3)扩大手动采集样本量（让研究者从APP手动复制评论）。

3. **非遗坐标定位到镇街中心而非具体传习所**：高德地理编码对非遗项目名称（如"狮舞（广东醒狮）西樵镇"）的理解有限，多数返回的是镇街中心坐标而非具体传习所的精确位置。优化方向：直接搜索传习所/传承基地的具体名称（如"佛山市南海黄飞鸿中联电缆武术龙狮协会"）进行定位。

4. **POI分类中"其他"类353条（26%）**：高德类型编码与文旅分类不完全匹配，部分POI无法自动分类。优化方向：引入人工校验或更精细的多级关键词匹配规则。

### 8.2 方法层面的不足

5. **体验度评估高级别为0**：五维评估模型的阈值设定过于严格。优化方向：(1)将"高"的阈值从70降低到60；(2)引入相对排名法（如前10%为高）替代绝对分值法；(3)增加"游客量""开放时间""配套设施"等维度。

6. **耦合分析依赖人工映射**：25对文化-景点映射关系由研究者手动建立，存在主观性和遗漏风险。优化方向：(1)基于文本相似度（如word2vec向量余弦相似度）自动匹配；(2)基于空间邻近性（3km缓冲区）自动关联；(3)引入专家评审校验。

7. **空间聚类参数敏感**：DBSCAN的eps=3km和min_points=3是经验值，不同参数会产生不同聚类结果。优化方向：使用轮廓系数(Silhouette Coefficient)进行参数自动调优。

8. **知识图谱仅150个节点**：为可视化可读性限制了节点数量，损失了大量实体信息。优化方向：(1)使用Neo4j图数据库存储完整图谱（2,279个实体+5,000条关系）；(2)在可视化中增加搜索和筛选功能；(3)按子图分层展示。

### 8.3 流程层面的不足

9. **缺少可达性分析**：开题报告中规划的"等时圈分析"（15/30/60分钟交通可达性）尚未实现。需调用高德路径规划API计算。

10. **缺少时序分析**：当前分析为静态截面，未利用评论的时间维度分析体验度变化趋势（如节假日 vs 工作日）。

11. **论文写作尚未开始**：中期阶段产出了完整的数据和分析结果，但尚未将其组织为论文正文的章节结构。
