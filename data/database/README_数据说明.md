# 数据库目录数据说明

本目录包含中期阶段所有结构化分析数据。每份数据的来源、获取方式、处理逻辑和参数如下。

---

## 1. culture_entities.json — 文化实体库

- **数据量**：626个纯文化实体（零"其他"类）
- **来源文本**：53个文本源，总计5,414,760个中文字符
  - 南海县志OCR连续文本（1个，约36.9万字）
  - 开题阶段典籍（22个.md文件，Z-Library获取，含佛山市志上下册、南海市文化艺术志、西樵山旅游度假区志等）
  - 南海文史资料（31个.txt文件，由36本扫描PDF经RapidOCR 12线程并行识别，排除5本涉政/已处理）
  - 每个文件加载上限500,000字符，超出部分截断
  - cultural_anchors.json 加载165条文化载体锚点到词典
- **处理脚本**：`code/data_processing/text_ner.py`
- **处理流程**：
  1. **停用词过滤**：200+常见功能词（如"我们""已经""可以""工作"等）在分词阶段过滤
  2. **jieba分词**：加载自定义词典（6类共300+个领域术语），设置词频1000防止被错误切分
  3. **实体识别（精确匹配）**：分词结果与自定义词典逐一比对，命中则标记对应类型，置信度0.95
  4. **实体识别（后缀规则）**：未命中词典的2-8字中文词，按后缀判定类型：
     - `*村/镇/区/市/县/街/路/山/岛/湖/江/河/洲/岗/塘/围/涌/窖/圩/墟` → 地点（0.8）
     - `*代/朝/年间/年`（≤4字）→ 朝代（0.8）
     - `*祠/庙/寺/院/堂/馆/楼/坊/塔/桥/亭/阁/庵/宫/殿/墓/庐/园/台/坛` → 建筑（0.8）
     - `*拳/舞/歌/曲/戏/艺/编/绣/塑/雕/陶/瓷/画/锻/染/织/酿` → 文化要素（0.75）
     - `*先生/公/氏/翁/侯/侍郎/尚书/知县/县令/总督/巡抚/提督/状元/太守`（2-5字）→ 人物（0.7）
  5. **过滤**：未分类实体直接丢弃（不再保留"其他"类）
  6. **TF-IDF计算**：每个实体在其来源文本中的TF-IDF值
  7. **跨源合并**：同名实体跨文本源合并，跨源权重 = 频次 × (1 + 0.3 × (源数-1))
- **自定义词典构成**（400+词，含anchor注入）：
  - 人物（65词）：黄飞鸿、康有为、叶问、戴鸿慈、陈启沅、詹天佑、石景宜、伦文叙、朱次琦等
  - 地点（80词）：行政区划、山水景观、古村落、历史地名等
  - 建筑（60词）：书院、宗祠、寺庙、纪念馆博物馆等
  - 文化要素（65词）：武术、音乐戏曲、传统工艺、饮食文化、民俗等
  - 事件（23词）：历史事件、文化活动等
  - 朝代（30词）：朝代名、年号等
- **类型分布**：地点206(32.9%)、建筑116(18.5%)、朝代114(18.2%)、人物95(15.2%)、文化要素88(14.1%)、事件7(1.1%)
- **关键字段**：name, type, weight, confidence, sources, tfidf_scores, is_anchor (boolean)

## 2. culture_relations.json — 文化实体关系库

- **数据量**：5,000条共现关系
- **处理脚本**：`code/data_processing/text_ner.py`（`extract_relations()`函数）
- **处理流程**：
  1. 将全部文本按标点分句（句号/感叹号/问号/分号/换行符作为分隔符）
  2. 对每个句子（≥10字），查找其中包含的已识别实体（每句最多取前6个）
  3. 同句中任意两个实体构成一对共现关系，累加共现次数
  4. 按共现频次降序排列，取频次≥2的前5,000对
- **字段**：entity1, entity2, co_occurrence（共现次数）, type1, type2（两端类型）

## 3. poi_cleaned.json — 标准化POI数据库

- **数据量**：13,113条
- **原始来源（双源融合）**：
  - 高德开放平台Web服务API（`https://restapi.amap.com/v3/place/text`）— 1,353条
  - 南海三镇文化资源普查Shapefile — 12,906条
  - 融合去重后共13,113条
- **采集脚本**：`code/data_collection/amap_real_data.py`
- **采集策略（双重检索）**：
  - **按类型编码检索**（15种）：旅游景点(110000)、风景名胜(110100)、公园广场(110101)、纪念馆(110205)、历史建筑(110206)、博物馆(140100)、展览馆(140200)、图书馆(140300)、科技馆(140400)、文化宫(140500)、影剧院(140600)、文化传媒(141200)、宗教场所(160000)、体育休闲(080000)等
  - **按关键词检索**（18个）："非遗""传习所""传承基地""古村""古镇""祠堂""庙宇""纪念馆""文化馆""遗址""醒狮""龙舟""武术""咏春""书院""故居""古建筑""历史建筑"
  - 每次请求限定 `city=佛山`、`citylimit=true`，二次过滤 `adname` 包含"南海区"
  - 原始返回1,590条 → 去重后1,353条（API部分）
- **清洗脚本**：`code/data_processing/poi_cleaner.py`
- **自动分类规则**（优先关键词匹配，其次高德类型编码二次分类，11类）：

  | 其他 | 1,901 | 未命中任何类别 |
  | 宗教场所 | 465 | 寺、庙、观、教堂 |
  | 人文古迹 | 366 | 古村、遗址、祠堂、牌坊、书院、文物 |
  | 文化场馆 | 286 | 博物馆、纪念馆、展览馆、文化馆、艺术馆 |
  | 休闲娱乐 | 237 | 影视城、度假、乐园、温泉 |
  | 体育设施 | 171 | 体育、运动、球场、健身 |
  | 非遗体验 | 135 | 非遗、传习所、传承基地、醒狮、咏春、藤编 |

- **新增字段**：cultural_anchors（关联的文化锚点名称）、has_cultural_anchor (boolean)、source (api/shapefile)
- **非遗关联匹配**：15组关键词，共164个POI关联到非遗项目
- **文化锚点关联**：996个POI关联到文化锚点

## 5. culture_taxonomy.json — 文化谱系分类原始数据

- **数据量**：8大类、24子类、97条目
- **处理脚本**：`code/analysis/culture_genealogy.py`
- **构建方式**：研究者综合以下三个来源编码：
  1. **非遗名录**（权威分类骨架）：南海区26项非遗的官方分类
  2. **典籍文本内容**（历史纵深补充）：从典籍中提炼的书院文化、科举传统、中医药文化等
  3. **NER实体频次**（数据验证排序）：用实体出现频次验证各条目的重要性
- **8大门类**：武术文化(3子类12条目)、饮食文化(3/10)、建筑文化(4/21)、民俗文化(2/11)、工艺文化(5/18)、音乐戏曲(2/6)、学术文化(3/7)、中医药文化(2/5)
- **关键人物**：黄飞鸿、康有为、叶问、湛若水、方献夫、霍韬、邹伯奇、戴鸿慈、陈启沅等15人

## 7. scenic_genealogy_tree.json — 景点谱系树

- **结构**：根节点 → 11个POI类别 → 体验度(高/中/低) → 具体景点
- **处理脚本**：`code/analysis/scenic_genealogy.py`
- **输入数据**：poi_cleaned.json（1,353条）+ review_summary_real.json（867条评论汇总）+ culture_entities.json（典籍提及度）
- **体验度评分公式**（六维加权）：

  | 维度 | 权重 | 计算方式 | 满分条件 |
  |------|------|---------|---------|
  | 平台评分 | 30% | (avg_rating / 5.0) × 100 | 评分5.0 |
  | 好评率 | 20% | positive_rate | 100%好评 |
  | 文化深度 | 20% | min(有非遗×30 + 非遗数×15, 100) | 关联多项非遗 |
  | 评论活跃度 | 15% | min(评论数 / 20, 1) × 100 | 20+条评论 |
  | 历史积淀 | 10% | 典籍文本提及次数归一化（/50封顶） | 高频提及 |
  | 照片丰富度 | 5% | min(照片数 / 15, 1) × 100 | 15+张照片 |

- **分级阈值**：≥60为高、≥40为中、<40为低
- **结果**：高=34、中=494、低=825

## 9. coupling_results.json — 双谱系耦合分析结果

- **处理脚本**：`code/analysis/coupling_analysis.py`
- **核心逻辑**：
  1. **定性分析**（四象限）：手动建立文化要素与景点之间的25对映射关系，判定四种耦合状态
  2. **定量分析**（耦合协调度）：按镇街计算文化资源密度(C)和旅游发展度(T)的协调关系

- **四种耦合状态**：
  - **强耦合**(16对)：匹配类型为"实体对应"或"文化承载"
  - **错位**(9对)：匹配类型为"空间共存"或"主题关联"
  - **缺失A**(14项)：非遗名录中无对应旅游产品
  - **缺失B**(6个)：景点缺乏文化内涵支撑

- **耦合协调度模型**：
  - T（旅游发展度）= 镇街POI数 / 最大POI数
  - C（文化资源度）= 非遗数/最大×0.6 + 典籍提及/最大×0.4
  - 耦合度 = sqrt(C×T) / ((C+T)/2)
  - 综合发展指数 = 0.5×C + 0.5×T
  - 耦合协调度 D = sqrt(耦合度 × 综合发展指数)
  - 分级：D≥0.8极高, D≥0.6高度, D≥0.4中度, D≥0.2低度, D<0.2极低

- **镇街耦合协调度结果**：

  | 镇街 | D值 | 等级 |
  |------|-----|------|
  | 桂城街道 | 0.880 | 极高协调 |
  | 西樵镇 | 0.759 | 高度协调 |
  | 大沥镇 | 0.704 | 高度协调 |
  | 里水镇 | 0.604 | 高度协调 |
  | 狮山镇 | 0.591 | 中度协调 |
  | 九江镇 | 0.500 | 中度协调 |
  | 丹灶镇 | 0.428 | 中度协调 |

## 数据流全景

```
文化典籍(54个文件, 541万字)
  ├── jieba+NER(400+词典+anchor注入+停用词) ──→ culture_entities.json (626个纯文化实体,零"其他"类)
  └── 共现分句 ──→ culture_relations.json (5,000条关系)
                        │
                        ├──→ culture_taxonomy.json (8类97条目)
                        │      └──→ culture_genealogy_tree.json (ECharts树)
                        │
                        └──→ expanded_knowledge_graph.json (387节点2,118边,三层锚定结构)
                                ↑ 融合(10类关系)
南海三镇Shapefile ──解析──→ cultural_anchors.json (165条文化载体锚点)
  ├── 注入NER词典 ──→ culture_entities.json (is_anchor标记)
  └── 12,906条POI ──┐
                     ├──融合去重──→ poi_cleaned.json (13,113条,关联锚点996个)
高德API ──双重检索──→ 1,353条原始POI ──┘
  │                 ├──→ scenic_genealogy_tree.json (六维体验度:高34/中494/低825)
  │                 ├──→ scenic_town_tree.json (镇街分布)
  │                 └──→ coupling_results.json (四象限+耦合协调度D)
  │                        ↑ 映射25对+协调度模型
  └── 行政区划 ──→ nanhai_boundary/towns.geojson

评论爬取(携程API+马蜂窝+高德评分) ──→ 948条评论/评分
  └── 情感分析 ──→ review_summary_real.json (867个景点汇总)
                        └──→ 六维体验度评分

非遗整理(26项) + 地理编码 ──→ nanhai_nonheritage.json/geojson
```
